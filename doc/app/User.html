    class User

    Validations

    court, email, name, password, password_confirmation, and role are all required.

    email

    Must be unique within the court.

    password

    At least six characters.

    role

    One of USER_ROLES, defined in config/initializers/site_config.rb.

    Cascading

    It destroyss dependent bookings. This means that CourtSessions that no longer have any reason_to_exist? are also destroyed.

    cancelled_bookings are deleted. There is really nothing to destroy.

def self.disabled_count( court = nil)
  if court
    where( "court_id = ? and role = ?", court, "disabled").count
  else
    where( "role = ?", "disabled").count
  end
end

def self.order_by_role_and_name( court)
  where( "court_id = ?", court.id).sort do |u1, u2|
    if u1.role == u2.role
      u1.name <=> u2.name
    else
      u1.role_to_order <=> u2.role_to_order
    end
  end
end
